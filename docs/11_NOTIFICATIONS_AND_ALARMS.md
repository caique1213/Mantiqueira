# 11 — Notificações e Alarmes

## Origem

O sistema anterior possuía alarme e fala.
A nova versão deve preservar a ideia, com UX melhor.

## Eventos

- nova OS elétrica;
- nova OS mecânica;
- OS crítica;
- OS atribuída ao usuário;
- OS aguardando peça;
- OS reaberta;
- falha recorrente detectada.

## Sons

Disponibilizar pelo menos 5 sons configuráveis.

Configuração:

- ligado/desligado;
- volume;
- som;
- repetição;
- fala opcional.

## Fala

Pode anunciar algo como:

> Nova OS crítica na Postura 27.

Deve respeitar:

- preferência do usuário;
- permissão;
- navegador;
- não repetir infinitamente.

## Acknowledgement

O usuário pode reconhecer o alerta.

## Não fazer

- tocar som a cada refresh;
- repetir a mesma OS indefinidamente;
- tocar alertas de setor não autorizado.
