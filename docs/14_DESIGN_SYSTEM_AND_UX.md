# 14 — Design System e UX

## Direção visual

Referência:

- interface industrial;
- escura;
- sofisticada;
- imagens grandes;
- pouco ruído;
- laranja/amarelo apenas como destaque;
- navegação simples;
- detalhes sob demanda.

As imagens disponíveis em `references/images/visual_atlas_parts/` são referência de atmosfera e qualidade, não conteúdo literal do novo sistema.

## Home

Deve ser limpa.

Sugestão:

- hero;
- pesquisa global;
- módulos principais;
- poucos indicadores;
- recentes/favoritos.

## Módulos principais

- Ordens de Serviço.
- Mapa das Posturas.
- Inventário Técnico.
- Análises.
- Administração.

## Cards

Usar quando agregam valor.

Evitar:

- excesso;
- bordas em tudo;
- nesting.

## Navegação

Desktop:

- header ou sidebar elegante.

Mobile:

- drawer/bottom nav conforme auditoria.

## Painéis laterais

Excelente para:

- detalhe de ativo;
- detalhe de postura;
- filtros.

## Modais

Usar somente quando o usuário precisa concluir uma tarefa curta.

## Bottom sheets

Preferir no mobile para detalhes contextuais.

## Animação

- suave;
- funcional;
- sem excesso.

## Estado de edição

Deve ser visualmente diferente.

Exemplo:

- banner "Modo edição ativo";
- ferramentas específicas;
- confirmação ao sair com alterações não salvas.

## Acessibilidade

- contraste;
- foco;
- tamanho de toque;
- navegação por teclado;
- labels.

## Imagens

- compressão;
- lazy loading;
- Storage privado;
- skeleton.

## Ícones

Consistentes.

## Feedback

Toda ação deve informar:

- salvando;
- sucesso;
- erro;
- permissão.

## Não usar

- mensagens genéricas demais;
- spinners infinitos;
- elementos que "somem" por falha;
- layout mobile espremido.
