# 07 — Ordens de Serviço

## Objetivo

Registrar, acompanhar e analisar falhas e intervenções.

## Abertura

Pode ocorrer por:

- botão Abrir OS;
- mapa;
- postura;
- bateria;
- ativo.

## Fluxo recomendado

### 1. Local

- Postura.
- Bateria.
- posição.

### 2. Setor

- Elétrica.
- Mecânica.

### 3. Ativo

Opcional, mas recomendado.

Se a OS for aberta a partir do ativo, preencher automaticamente.

### 4. Tipo de problema

Exemplos configuráveis:

- esteira de nylon;
- esteira branca;
- elevador;
- ração;
- exaustor;
- iluminação;
- motor;
- redutor;
- sensor;
- outros.

### 5. Descrição

Texto livre.

### 6. Prioridade

Padrão:

- Baixa.
- Normal.
- Alta.
- Crítica.

### 7. Anexos

- foto do problema;
- vídeo futuro opcional;
- documento futuro opcional.

## Status padrão

- Aguardando atendimento.
- Em execução.
- Aguardando peça.
- Resolvida.
- Cancelada.

## Atendimento

Ações:

- assumir;
- iniciar;
- adicionar observação;
- registrar diagnóstico;
- registrar causa;
- registrar serviço;
- anexar foto;
- aguardando peça;
- resolver;
- cancelar conforme permissão.

## Aguardando peça

Registrar:

- peça necessária;
- descrição;
- código;
- fabricante;
- quantidade estimada;
- observação.

Isso não representa saldo de almoxarifado.

## Finalização

Registrar:

- diagnóstico final;
- causa;
- serviço realizado;
- observação;
- data;
- responsável;
- fotos depois.

## Bloqueio

OS finalizada:

- sem botões operacionais comuns;
- histórico visível;
- reabertura somente com fluxo específico.

## Histórico da OS

Linha do tempo:

- abertura;
- mudança de prioridade;
- responsável;
- status;
- comentários;
- anexos;
- conclusão;
- reabertura.

## Painéis setoriais

### Elétrica

Somente o necessário ao setor.

### Mecânica

Somente o necessário ao setor.

Admin vê ambos.

## Relatório diário

Exportar OS por data.

Filtros:

- data;
- setor;
- postura;
- responsável;
- status;
- prioridade.

## Busca

Buscar por:

- número;
- postura;
- bateria;
- ativo;
- marca;
- modelo;
- descrição;
- responsável.
